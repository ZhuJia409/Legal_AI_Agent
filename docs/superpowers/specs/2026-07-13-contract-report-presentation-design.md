# 合同审查报告版式与下载界面精简设计

## 目标

- 修正 PDF 封面任务信息的左右对齐关系。
- 用户、模型和 LangSmith 可见内容不再出现旧阶段编号，统一使用“合同背景审查”或“背景审查”。
- 删除 PDF 末尾独立的“免责声明”章节。
- 前端完整合同审查成功后只提供 PDF 下载交付界面，不展示报告正文详情。
- 保留必要的法律专业人士复核提示和 partial 报告警示。

## PDF 封面

封面任务信息继续使用两列表格，不增加新的 LaTeX 宏包。表格宽度占满正文区域：

- 左列为任务编号、报告状态、审查立场、原始文件、生成时间，统一左对齐。
- 右列为对应值，统一右对齐。
- 右列使用固定宽度段落列和 `\raggedleft\arraybackslash`，长文件名可以自动换行，不得超出页面。
- 每行上下间距保持一致，不改变标题、状态色和 partial 警告框。

## 报告章节与文案

- 背景审查章节改为“二、合同背景审查”，目录自动使用新标题。
- 章节提示改为“背景审查提示”。
- 删除“九、免责声明”标题及其全部正文；“八、审查限制”成为最后一个报告章节。
- 封面底部继续保留“法律审查辅助信息”和“法律专业人士复核”提示，以满足项目法律输出规范。
- `ContractReviewReportResponse.disclaimer` 字段继续保留，避免改变 API schema 和旧快照兼容性，但 PDF 和精简前端不再展示该字段。

## 旧阶段编号清理边界

以下用户、模型或审计人员可见位置必须改为“合同背景审查”或语义一致的“背景审查”：

- PDF 模板、前端界面、后端公开文案和错误说明。
- LLM system/user prompt、证据上下文提示和 LLM 工具名称/说明。
- LangSmith run name、tag、metadata、LangGraph 节点名称及模块日志名称。
- 项目 README、AGENTS 业务说明和代码注释/docstring。

为保障兼容性，以下纯内部历史标识允许保留：

- 已发布 Alembic 迁移文件名、revision 内容及既有数据库结构。
- 不进入模型、报告、前端或 LangSmith 的内部常量名；本次触及的内部函数和常量可同步改为 `background_review` 命名，但不为追求全仓字符串为零而改写历史迁移。
- 用户未跟踪的解析样例目录不在本次修改范围内。

验收时分别执行“运行时代码可见内容扫描”和“历史兼容标识清单检查”，不得把历史迁移误判为公开文案残留。

## 前端下载交付页

`ContractReviewReportResult` 不再渲染背景卡、合同类型、模块状态、风险等级、结论、风险事项、签署前提、证据限制和响应 disclaimer。

成功结果仅包含：

1. partial 时显示“不完整报告 / 不可作为签署依据”警示；complete 不额外显示成功横幅。
2. PDF 文件卡，展示文件名、大小、上海时区生成时间和下载/重试按钮。
3. 一条克制的法律专业人士复核提示。

旧任务的 `report_document` 为空时，显示简洁的“PDF 报告文件暂不可用，请重新发起审查”状态，不回退展示完整报告正文。下载仍通过 Next.js 同源 Route Handler，不暴露后端或 MinIO 地址。

## 测试与验收

### 后端与 PDF

- 模板测试断言封面左列左对齐、右列右对齐并允许换行。
- 目录和报告正文只使用“合同背景审查”，不包含旧阶段编号。
- 报告不包含“九、免责声明”及固定免责声明正文。
- 封面仍包含法律专业人士复核提示。
- 提示词与运行时代码可见文案扫描不包含旧阶段编号。
- 真实 Tectonic 生成 PDF 后转为图片，人工核对封面、目录和末页。

### 前端

- 组件测试或静态契约测试确认结果组件只组合 partial 警示、文件卡、不可用状态和法律复核提示。
- TypeScript 类型保持 `report_document` 可空兼容。
- 运行 lint、typecheck 和 build。
- 浏览器检查桌面端和移动端下载卡布局，不出现报告正文详情。

### 完整命令

- `cd backend; uv run ruff check .`
- `cd backend; uv run pytest`
- `cd frontend; pnpm.cmd lint`
- `cd frontend; pnpm.cmd typecheck`
- `cd frontend; pnpm.cmd build`
- `git status --short`

## 非目标

- 不删除 API 响应中的结构化审查字段。
- 不修改数据库表、快照结构或历史 Alembic 迁移。
- 不增加 PDF 内嵌预览、在线编辑或重新生成按钮。
- 不改变下载接口、安全响应头和 MinIO 对象结构。
