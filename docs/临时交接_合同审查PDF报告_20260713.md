- `backend/tests/test_contract_review_pdf.py`
- `scripts/setup-tectonic.ps1`

已实现：

- 异步 `TectonicCompiler`，使用 `asyncio.create_subprocess_exec`，不使用 shell。
- 独立临时目录，命令固定包含：
  `--only-cached --keep-logs --outdir <temp> report.tex`。
- 缺少或无法启动 Tectonic 时抛出 `PdfRendererUnavailableError`，用于 API 映射 503。
- 非零退出、超时、空输出、非 PDF 输出统一为 `ReportPdfGenerationError`。
- 超时、请求取消和通信异常都会在进程仍存活时执行 `kill + await wait`，避免 Windows 文件占用和孤儿进程。
- stderr 原文完全不进入异常，避免合同内容泄露。
- setup 和运行时统一使用仓库级缓存：
  `.tools/tectonic/cache`，通过 `TECTONIC_CACHE_DIR` 注入，不依赖安装用户的个人缓存。
- 默认 Tectonic 0.15.0 官方 Windows 包和固定 SHA-256：
  `1D6BB76F049C8A3774F6E9D66E4B04E1A8C3DCB37527B6B41B7E894328E7BF29`。
- 安装脚本支持显式覆盖 Version、DownloadUrl、ExpectedSha256。
- 安装脚本先联网预热中文模板及宏包，再删除产物，使用 `--only-cached` 离线复编译验证。
- 正式 exe 采用同目录 staged 文件、SHA 校验和 `File.Replace`/`File.Move` 原子切换，失败不破坏已有版本。
- PowerShell 脚本使用 UTF-8 BOM，兼容 Windows PowerShell 5.1。

## 6. 尚未完成的工作

### 6.1 后端 PDF 集成

尚未开始，需按 TDD 完成：

1. 增加 PDF renderer 依赖组装，读取 `Settings.tectonic_path` 和 timeout。
2. `POST /api/v1/contract-review-reports` 在 Phase 2 后生成 PDF。
3. 将 `generated.to_document_info(task_id)` 回填到 `result.report_document`。
4. 扩展 `ContractReviewPersistenceService.persist_review`，在同一次持久化调用中保存 PDF：
   - 对象 key：`contract-reviews/{task_id}/reports/<filename>`。
   - `document_type="contract_review_report_pdf"`。
   - snapshot 的 `full_report` 必须包含 `report_document`。
5. 扩展对象存储协议，支持读取 bytes；MinIO 读取失败转换为受控错误。
6. 扩展 repository，支持按 task ID 和 document type 查询文档元数据。
7. 新增下载接口：
   `GET /api/v1/contract-review-reports/{task_id}/document`。
8. 下载接口要求：UUID 校验、中文 RFC 5987 文件名、`Content-Type`、`Content-Length`、SHA-256 ETag、`Cache-Control: private, no-store`、`X-Content-Type-Options: nosniff`。
9. 错误映射：
   - 缺少 Tectonic：503 `pdf_renderer_unavailable`。
   - 编译失败或超时：500 `report_pdf_generation_error`。
   - 文档不存在：404 `report_document_not_found`。
   - MinIO 读取失败：503 `report_document_read_error`。
10. Phase 0 接口不得生成 PDF，行为保持不变。

建议先拆为两个顺序任务：

- A：对象存储、repository、persistence 的 PDF 写入/查询/读取。
- B：POST 渲染编排和 GET 下载 API。
