# 33. 数据处理协议（DPA）规则包

- 法律依据：《个人信息保护法》；《数据安全法》；《网络安全法》；《民法典》第1034-1039条；GDPR（如涉及欧盟数据）。
- 适用场景：委托处理、共同处理、数据跨境、个人信息处理、重要数据处理等。
- 路由识别特征：出现“个人信息、数据处理、委托处理、共同处理、处理目的、数据出境、删除返还、再委托、DPA”等表达。
- 必查条款：处理目的、处理范围、处理期限、处理方式、双方角色、个人信息主体权利、安全措施、再委托、跨境传输、删除返还、审计权、事件通知。
- 专项审查重点：检查 PIPL 第21条委托处理要求，共同处理责任分配，跨境合规路径，敏感个人信息单独同意，安全事件通知和协助义务。
- 特殊陷阱：主合同涉及个人信息但未签 DPA；处理目的过宽；再委托无限制；数据出境无合规路径；罚则分担和赔偿上限不匹配。
- 输出给主智能体的风险项：DPA 缺失风险、处理范围过宽风险、再委托风险、数据出境风险、敏感信息处理风险、安全事件责任风险。
