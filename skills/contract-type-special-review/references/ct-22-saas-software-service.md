# 22. SaaS / 软件服务协议规则包

- 法律依据：《民法典》合同编；《网络安全法》；《数据安全法》；《个人信息保护法》；软件和知识产权相关法律。
- 适用场景：云软件订阅、软件许可、在线服务、实施服务、运维服务、SLA 服务。
- 路由识别特征：出现“SaaS、软件服务、订阅、账号、SLA、云服务、数据、接口、实施、运维、服务可用性”等表达。
- 必查条款：服务范围、账号权限、服务期限、费用、SLA、数据权属、个人信息处理、数据安全、系统变更、服务暂停、退出迁移、知识产权。
- 专项审查重点：检查 SLA 是否量化，服务中断责任和赔偿上限，数据归属和导出，数据处理角色，续费和自动续订，开源和第三方组件。
- 特殊陷阱：供应商单方变更或停服权过宽；数据导出和删除缺失；赔偿上限远低于业务损失；SLA 只有宣传无承诺；个人信息处理缺少 DPA。
- 输出给主智能体的风险项：SLA 不足风险、数据权属风险、停服变更风险、赔偿上限风险、个人信息合规风险、退出迁移风险。
