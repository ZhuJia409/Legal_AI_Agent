# 24. 采购框架协议规则包

- 法律依据：《民法典》合同编；招投标、政府采购或企业采购制度视场景适用。
- 适用场景：长期采购、框架采购、订单采购、主协议加订单/SOW 的交易结构。
- 路由识别特征：出现“框架协议、采购、订单、报价单、SOW、采购需求、下单、交付、价格清单”等表达。
- 必查条款：框架与订单效力层级、采购范围、价格机制、下单流程、交付验收、付款结算、最低采购量、变更、附件和 SOW、终止。
- 专项审查重点：检查框架协议是否直接产生采购义务，订单和附件优先级，价格调整机制，背靠背付款，最低采购承诺，招投标文件一致性。
- 特殊陷阱：框架协议被误认为无约束但已约定实质义务；订单与主协议冲突；最低采购量不清；背靠背付款转嫁上游风险；附件缺失导致无法履行。
- 输出给主智能体的风险项：文件效力层级风险、订单冲突风险、价格调整风险、最低采购量风险、背靠背付款风险、附件缺失风险。
